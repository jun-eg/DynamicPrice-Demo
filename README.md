# DynamicPrice-Demo

旅館向けダイナミックプライシングのデモプロジェクト。

## 概要

旅館の客室・プランに対して、需要・在庫・季節性などをもとに価格を動的に算出する仕組みを試作する。

設計方針・決定の経緯・運用手順はリポジトリ内のドキュメントを唯一の真実とする。

## ドキュメント

| フォルダ             | 問い                                         |
| -------------------- | -------------------------------------------- |
| `docs/architecture/` | なぜこの設計か（原則・制約・思想）           |
| `docs/adr/`          | なぜこの決断をしたか（判断の記録・追記のみ） |
| `docs/runbooks/`     | どう操作するか（手順・コマンド）             |

## ブランチ戦略

- `main`: 本番相当
- `develop`: 開発統合
- `feature/*`: 機能開発

`main` / `develop` への直接 push は禁止（PR 必須）。

## ステータス

設計フェーズ。技術スタック未確定。
