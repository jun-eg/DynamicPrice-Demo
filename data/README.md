# data/

実データの置き場。ディレクトリ構造のみ git 管理し、ファイル実体は管理外。

## 構成

| パス               | 内容                                       | git管理 |
| ------------------ | ------------------------------------------ | ------- |
| `data/raw/`        | 元CSV(無加工)                              | 除外    |
| `data/processed/`  | 加工済み(Parquet/DuckDB等)                 | 除外    |
| `data/sample.csv`  | 匿名化したサンプル(数十行、テスト・デモ用) | 含む    |

## 配置ルール

- 実データは各自ローカルで `data/raw/` に置く
- ファイル名は固定する(後でスクリプトから読むため) → `docs/runbooks/data-setup.md` 参照(未作成)
- 個人情報を含むデータをコミットしないこと
- スキーマ定義は `docs/architecture/data-model.md` に記載(未作成)

## なぜ git に入れないか

`docs/adr/0002-data-management.md` 参照(未作成)。要点:

- サイズ: 履歴肥大化を避ける
- プライバシー: 予約データは個人情報を含む
- 再現性: スクリプト(ETL)で担保し、データそのものは外部管理
